**日期：** 2026-05-26
**状态：** 已确认

---

## 一、概述

**工具名称：** ConfigGen
**用途：** 通过模板 + Excel 数据生成配置文件，适用于批量生成相似配置
**典型场景：** 36 家分行的 nginx / Spring Cloud Gateway 配置

**用户画像：**
- 运维工程师
- 需要批量生成相似但参数不同的配置文件
- 不需要复杂的 CI/CD 流程

---

## 二、技术方案

### 2.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Spring Boot 3.x | 单 JAR 打包，内嵌 Tomcat |
| 前端 | Vue.js 3 + Element Plus | 嵌入式 SPA，无需单独部署 |
| 编辑器 | Monaco Editor | VS Code 同款，支持语法高亮 |
| 构建 | Maven + jpackage | JAR 跨平台，exe 原生安装 |

### 2.2 部署形态

| 分发形式 | 用户体验 |
|---------|---------|
| `config-gen.jar` | `java -jar config-gen.jar` 运行，Mac/Windows/Linux 通用 |
| `config-gen.exe` | Windows 双击运行（jpackage 打包，内置 JRE） |

**默认端口：** `http://localhost:8080`，启动后自动打开浏览器。

### 2.3 模块架构

## 三、功能设计

### 3.1 模板管理

**功能清单：**
- 上传模板文件（.conf / .yaml / .properties 等）
- 在线编辑模板（Monaco Editor，语法高亮）
- 内置示例模板：nginx、Spring Cloud Gateway 各一份
- 自动识别 `{{字段名}}` 占位符，提取字段列表

**模板语法：** `{{字段名}}`（双花括号）

**理由：** Nginx / Gateway 原生语法均不使用双花括号，不会产生混淆。

**字段展示：**
```
检测到 3 个占位符：
{{port}}、{{domain}}、{{upstream}}

请确保 Excel 中包含对应的列
```

**模板操作：**
- 新建空白模板
- 复制已有模板
- 下载模板文件
- 不持久化，关闭页面后清除

### 3.2 Excel 数据管理

**功能清单：**
- 支持 .xlsx、.csv 格式
- 上传后立即解析，提取列名与所有数据行
- 列名与模板占位符自动匹配校验
- 表格展示所有数据行，标记匹配状态
- 行数统计

**字段匹配展示：**
```
Excel 解析成功！
├─ 字段：port、domain、upstream
├─ 数据行：36 行
├─ 匹配状态：
│   ✓ port — 36 个值
│   ✓ domain — 36 个值
│   ✓ upstream — 36 个值
└─ 所有字段已匹配 ✓
```

**文件名列：** 固定使用 `filename` 列指定输出文件名。

**数据操作：**
- 重新上传 Excel（覆盖）
- 下载示例 Excel（含字段名 + 2 行示例数据）

### 3.3 配置生成与预览

**填充时机：** 上传 Excel 后立即用每行数据填充模板，结果存入内存。

**预览功能：**
- 左侧列表展示所有数据行（如 36 家分行）
- 点击某行，右侧实时显示该行填充后的完整配置
- 语法高亮（根据模板类型）
- 填充值高亮显示

**预览界面布局：**
```
┌─────────────────┬──────────────────────────────┐
│  分行列表        │  配置预览                      │
│  ┌───────────┐  │  ┌──────────────────────────┐  │
│  │ 北京分行  │  │  │ server {                 │  │
│  │ 上海分行  │  │  │   listen 8080;           │  │
│  │ 广州分行 ←───  │  │   server_name bj.test;   │  │
│  │ 深圳分行  │  │  │   ...                    │  │
│  │ ...      │  │  │ }                        │  │
│  └───────────┘  │  └──────────────────────────┘  │
│  [下载 ZIP]     │                              │
└─────────────────┴──────────────────────────────┘
```

### 3.4 打包下载

**ZIP 打包：**
- 每个配置文件以 Excel `filename` 列的值命名
- 文件格式保持模板原始扩展名
- 压缩包名：`configs-{{时间戳}}.zip`

**下载示例：**
```
configs-20250526-143052.zip
├── beijing-nginx.conf
├── shanghai-nginx.conf
├── guangzhou-nginx.conf
└── ...
```

## 四、数据流

```
1. 用户上传模板 → 系统识别 {{字段名}} → 提取字段列表
2. 用户上传 Excel → 系统解析列名与数据行
3. 列名与占位符匹配校验 → 如不匹配则提示警告
4. 匹配成功 → 立即用每行数据填充模板 → 结果存内存
5. 用户点击预览 → 选中某行 → 显示该行填充结果
6. 用户点击下载 → 内存中所有填充结果打包 ZIP → 浏览器下载
```

## 六、约束与边界

- **不持久化**：模板、Excel 数据关闭页面后清除，不做存储
- **不支持条件逻辑**：所有分行用同一套逻辑，仅变量值不同
- **不支持循环**：每行数据生成一个配置文件，一对一映射
- **文件名字段固定**：必须有 `filename` 列
- **极简依赖**：纯前端工具，无需数据库，无需外部服务

## 七、内置模板示例

### 7.1 Nginx 模板

```nginx
server {
    listen {{port}};
    server_name {{domain}};

    location / {
        proxy_pass http://{{upstream}};
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**占位符：** `port`、`domain`、`upstream`、`filename`（文件名）

### 7.2 Spring Cloud Gateway 模板

```yaml
spring:
  cloud:
    gateway:
      routes:
        - id: {{route_id}}
          uri: http://{{upstream}}
          predicates:
            - Path=/{{path_prefix}}/**
          filters:
            - StripPrefix=1
```

**占位符：** `route_id`、`upstream`、`path_prefix`、`filename`（文件名）

### 7.3 示例 Excel

| filename | port | domain | upstream |
|----------|------|--------|----------|
| beijing-nginx.conf | 8080 | bj.test.com | beijing-backend |
| shanghai-nginx.conf | 8080 | sh.test.com | shanghai-backend |

## 八、验收标准

1. 单 JAR 文件，双击或 `java -jar` 即可运行
2. 浏览器自动打开本地界面
3. 上传模板后正确识别 `{{字段名}}` 占位符
4. 上传 Excel 后正确解析列名并匹配占位符
5. 点击任意一行可预览填充后的配置内容
6. 下载 ZIP 包含所有配置文件，文件名以 `filename` 列为准
7. 不持久化任何数据，关闭即清除
